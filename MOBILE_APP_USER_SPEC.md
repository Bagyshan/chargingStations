# Мобильное приложение «BatEnergy» — спецификация для пользователя

> Документ-бриф для ИИ-чата, который сгенерирует каркас и дизайн мобильного приложения (Android/iOS).
> Здесь описан **только клиент для конечного пользователя** (водитель электромобиля). Админка, панель оператора и SteVe-менеджер — вне рамок этого документа.
> Фокус: **как выглядят экраны, какие на них элементы и кнопки, какие переходы**. Логика контроллеров/эндпоинтов — отдельно (см. краткий список в конце), детально расписывать не нужно.

---

## 1. О продукте

Приложение для поиска зарядных станций для электромобилей, бронирования коннектора, запуска/остановки зарядки в реальном времени и оплаты через предоплаченный кошелёк.

**Целевой пользователь:** владелец электромобиля, который хочет:
1. найти ближайшую свободную станцию на карте;
2. убедиться, что нужный коннектор свободен и подходит его авто;
3. забронировать/начать зарядку;
4. видеть процесс зарядки в реальном времени (кВт·ч, мощность, стоимость);
5. оплачивать с баланса кошелька и пополнять его.

**Платформы:** Android и iOS (один общий UX, нативный вид под каждую платформу).
**Язык интерфейса:** русский (заложить возможность локализации).

## 2. Глоссарий (термины в UI)

| Термин | Что это для пользователя |
|---|---|
| **Станция** | Физическая зарядная точка на карте (может иметь несколько коннекторов). |
| **Коннектор** | Конкретный разъём станции (Type 2, CCS, CHAdeMO и т.п.) со своей мощностью и статусом. |
| **Статус** | Состояние: свободен / занят / недоступен (сломан/отключён). Показывается цветом. |
| **Бронь** | Временное закрепление коннектора за пользователем перед зарядкой. |
| **Сессия зарядки** | Активный процесс зарядки с живыми показаниями. |
| **Кошелёк / Баланс** | Предоплаченные средства, с которых списывается стоимость зарядки. |
| **Пополнение (Top-up)** | Добавление денег на баланс (оплата через O!Dengi, QR/redirect). |
| **Тариф** | Стоимость зарядки (почасовая / за кВт·ч), привязана к станции. |
| **Избранное** | Станции, отмеченные пользователем для быстрого доступа. Несут те же данные, что и в общем списке (расстояние, статусы, тариф). |

## 3. Навигация (общая структура)

**Нижняя панель (bottom navigation), 4 вкладки:**

```
[ 🗺 Карта ]   [ ⚡ Зарядка ]   [ 💳 Кошелёк ]   [ 👤 Профиль ]
```

- **Карта** — главный экран, поиск и выбор станции.
- **Зарядка** — активная сессия (если идёт) или последняя/история. Бейдж-индикатор, когда зарядка активна.
- **Кошелёк** — баланс, пополнение, история.
- **Профиль** — данные пользователя, настройки, выход.

**Избранные станции** не выносятся в отдельную вкладку (чтобы не раздувать нижнюю панель). Вход в «Избранное» — иконка ⭐/♡ в верхней панели экрана **Карта** (быстрый доступ) и пункт в **Профиле**. Сам экран — отдельное окно (см. 4.17).

**Поток до входа** (без нижней панели): Splash → Онбординг → Регистрация/Вход → подтверждение email → внутрь приложения.

```
Splash
 └─(нет токена)→ Вход ──→ Карта (главный)
 └─(есть токен)→ Карта (главный)

Вход ↔ Регистрация ↔ Восстановление пароля
Регистрация → «Проверьте почту» → (deep link) → Карта
```

## 4. Экраны

### 4.1. Splash / автологин
- Логотип по центру, индикатор загрузки.
- Логика: есть валидный токен → сразу Карта; иначе → Вход.
- Без кнопок.

### 4.2. Онбординг (опционально, 2–3 слайда)
- Иллюстрации + короткий текст: «Найди станцию», «Заряди за пару касаний», «Плати с кошелька».
- Кнопки: **Далее**, **Пропустить**, на последнем слайде — **Начать**.

### 4.3. Регистрация
- Поля: **Email**, **Пароль** (с индикатором надёжности и кнопкой «показать»), **Имя**, **Фамилия**, **Телефон** (опционально).
- Чекбокс согласия с условиями.
- Кнопка **Зарегистрироваться** (основная, на всю ширину).
- Ссылка внизу: «Уже есть аккаунт? **Войти**».
- Состояния: валидация полей в реальном времени, ошибка «email уже занят», лоадер на кнопке.

### 4.4. Экран «Проверьте почту» (после регистрации)
- Иллюстрация конверта, текст: «Мы отправили ссылку на {email}. Подтвердите почту, чтобы войти».
- Кнопки: **Открыть почту**, **Отправить письмо повторно** (с таймером-кулдауном), «Изменить email».
- Подтверждение происходит по ссылке из письма (deep link / переход в браузер → возврат). После подтверждения — автоматический вход или экран успеха.

### 4.5. Вход
- Поля: **Email**, **Пароль** («показать»).
- Кнопка **Войти**.
- Ссылки: «**Забыли пароль?**», «Нет аккаунта? **Регистрация**».
- Состояния/ошибки: «неверные данные», «email не подтверждён» (с кнопкой «отправить письмо повторно»).

### 4.6. Восстановление пароля
- Шаг 1: поле **Email** + кнопка **Отправить ссылку**.
- Шаг 2 (по ссылке из письма): поля **Новый пароль** / **Повтор пароля** + кнопка **Сохранить**.
- Экран успеха → переход на Вход.

---

### 4.7. Карта станций — ГЛАВНЫЙ ЭКРАН
Самый важный экран. Полноэкранная карта.

**Элементы:**
- **Карта** с пинами станций. Цвет пина = статус:
  - 🟢 зелёный — есть свободные коннекторы;
  - 🟠 оранжевый — все заняты;
  - ⚪️ серый — недоступна (сломана/отключена).
- **Кнопка геолокации** (центрировать на пользователе).
- **Строка поиска** сверху (поиск по адресу/названию).
- **Кнопка «Фильтры»** → шторка фильтров: тип коннектора, минимальная мощность, только свободные, только доступные.
- **Кнопка «Список»** (переключение карта ↔ список).
- **Иконка «Избранное» ⭐** в верхней панели → переход на экран избранных станций (4.17).
- **Нижняя выезжающая карточка (bottom sheet)** при тапе на пин: мини-превью станции (название, адрес, расстояние, сколько коннекторов свободно, тариф), **иконка-сердечко ♡/♥** для добавления/удаления из избранного + кнопки **Подробнее** и **Маршрут**.

**Состояния:** загрузка станций, нет доступа к геолокации (подсказка включить), пусто в радиусе.

**Реалтайм:** статусы станций/коннекторов обновляются «вживую» (см. раздел 5) — цвет пина меняется без перезагрузки.

### 4.8. Список станций рядом (альтернатива карте)
- Список карточек, отсортирован по расстоянию.
- Карточка: иконка статуса, название, адрес, расстояние, «свободно X из Y», тариф, **сердечко ♡/♥** (избранное).
- Тап → Карточка станции. Кнопка переключения обратно на карту.

### 4.9. Карточка станции (детали)
**Элементы:**
- Шапка: название, адрес, индикатор статуса, расстояние, **иконка-сердечко ♡/♥** (добавить/убрать из избранного), кнопка **Маршрут** (в навигатор).
- (опц.) Фото/иллюстрация станции.
- **Список коннекторов** — для каждого: тип (иконка разъёма), мощность (кВт), статус (свободен/занят/недоступен). Свободные — выделены и кликабельны.
- Блок **Тариф** (стоимость зарядки).
- Главные кнопки (внизу, sticky):
  - **Забронировать** (если выбран свободный коннектор);
  - **Начать зарядку** (если пользователь уже на станции / по QR);
  - (опц.) **Сканировать QR** коннектора.
- Состояния: коннектор стал занят, пока смотрел (живое обновление + тост).

### 4.10. Бронирование
- Подтверждение: какая станция, какой коннектор, ориентировочная стоимость/условия.
- Проверка баланса: если средств мало — подсказка «Пополните кошелёк» + кнопка **Пополнить**.
- Кнопка **Подтвердить бронь**.
- После брони: экран/баннер «Коннектор забронирован» + **таймер** действия брони + кнопка **Отменить бронь** и **Перейти к зарядке**.
- Ошибки: «коннектор уже занят», «недостаточно средств», «таймаут» (бронь делается через сагу — возможен отказ, нужно дружелюбно показать).

### 4.11. Активная сессия зарядки ⚡ (ключевой реалтайм-экран)
Открывается при старте зарядки; доступна из вкладки «Зарядка».

**Элементы:**
- Крупный **круговой индикатор/анимация** процесса зарядки.
- Живые показатели (обновляются в реальном времени):
  - **кВт·ч** заряжено;
  - **текущая мощность** (кВт);
  - **время** сессии;
  - **накопленная стоимость** (₽/сом) и **остаток баланса**.
- Инфо о станции/коннекторе.
- Кнопки:
  - **Остановить зарядку** (основная, с подтверждением);
  - **Пополнить во время зарядки** (top-up «на лету» — продлевает активную сессию; см. бизнес-логику).
- Предупреждения: «низкий баланс — зарядка скоро остановится», «станция сообщила об ошибке/неисправности».
- Завершение → экран **Итог сессии** (заряжено кВт·ч, длительность, списано, новый баланс) + кнопка **Готово**.

### 4.12. Кошелёк / Баланс 💳
- Крупно: **текущий баланс**.
- Кнопка **Пополнить** (основная).
- Блоки: **История пополнений**, **История зарядных сессий** (с суммами).
- Тап на запись → детали.

### 4.13. Пополнение (Top-up через O!Dengi)
- Ввод **суммы** (поле + быстрые кнопки: 100 / 300 / 500 / 1000).
- Кнопка **Оплатить**.
- Оплата через O!Dengi: показ **QR-кода** и/или переход на платёжную страницу; экран ожидания со **статусом** (ожидание → успех/ошибка), polling статуса заказа.
- Экран успеха: «Баланс пополнен на N» + новый баланс, кнопка **Готово**.
- Обработать возврат из внешней оплаты обратно в приложение (deep link).

### 4.14. История
- Вкладки/фильтр: **Зарядки** и **Платежи**.
- Карточки записей: дата, станция (для зарядки), кВт·ч, сумма, статус.
- Тап → детальная карточка.

### 4.15. Профиль 👤
- Данные: имя, фамилия, email (с бейджем «подтверждён»), телефон (с бейджем верификации).
- Кнопки: **Редактировать профиль**, **Подтвердить телефон** (если не подтверждён), **Сменить пароль**.
- Пункт меню **Избранные станции ⭐** → экран 4.17.
- Раздел настроек: уведомления, язык, тема.
- Кнопка **Выйти** (внизу).

### 4.16. Редактирование профиля
- Поля: имя, фамилия, телефон.
- При смене телефона — флоу верификации (запрос кода → ввод кода).
- Кнопка **Сохранить**.

### 4.17. Избранные станции ⭐ (отдельное окно)
Быстрый доступ к станциям, которые пользователь отметил сердечком. Вход — с экрана **Карта** (иконка ⭐ в верхней панели) и из **Профиля**.

**Важно:** карточки избранного выглядят и ведут себя **точно так же**, как карточки в общем списке станций (4.8) — те же поля: статус (цвет), расстояние от текущей геопозиции, «свободно X из Y», тариф, и **тот же реалтайм** (цвет/доступность обновляются вживую). Это не «замороженный снимок» — данные подтягиваются из того же кэша станций, что и общий список.

**Элементы:**
- Заголовок «Избранное» + кнопка «назад».
- Список карточек избранных станций, отсортирован по расстоянию (как в 4.8).
- На каждой карточке: **залитое сердечко ♥** — тап убирает из избранного (с возможностью «Отменить»/undo).
- Тап на карточку → **Карточка станции** (4.9).
- (опц.) переключатель **карта ↔ список** — показать избранные пинами на карте.

**Состояния:**
- **Пусто:** иллюстрация + текст «Пока нет избранных станций. Добавляйте станции сердечком на карте» + кнопка **На карту**.
- Загрузка (скелетоны), нет сети / ошибка с кнопкой «Повторить».
- Станция из избранного больше недоступна (сломана/удалена) — показать серой с пометкой, не скрывать молча.

**Поведение данных (для разработчика):** приложение хранит у себя set ID избранного (после входа тянет один раз), чтобы мгновенно рисовать состояние сердечка на любой карточке (карта/список/детали). Полные данные станций для этого экрана запрашиваются по списку ID у кэширующего сервиса вместе с координатами пользователя — ответ идентичен общему списку станций.

## 5. Реальное время и статусы

Приложение получает живые обновления (через WebSocket-подключение к бэкенду):
- изменение **статуса коннекторов/станций** → цвет пинов и карточек обновляется мгновенно;
- **показания активной зарядки** (кВт·ч, мощность, стоимость) → обновление экрана сессии;
- **уведомления** о неисправности станции / завершении зарядки.

**Цветовая семантика статусов (единая по всему приложению):**
- 🟢 Свободно / Доступно
- 🟠 Занято / Идёт зарядка
- ⚪️/🔴 Недоступно (сломано/отключено) — нельзя выбрать, показывать причину.

**Пустые/ошибочные состояния — обязательны для каждого экрана:** загрузка (скелетоны), пусто, нет сети, ошибка с кнопкой «Повторить».

## 5a. WebSocket-соединение (реальное время) — техническая спецификация

Живые обновления приходят через одно постоянное WebSocket-соединение с `websocket-service`.

### Подключение
- **URL (через прод-домен/nginx → api-gateway):** `ws(s)://<host>/websocket/ws/station-events?token=<JWT>`
  - локально напрямую к сервису: `ws://localhost:8003/ws/station-events?token=<JWT>`
  - за nginx/HTTPS использовать `wss://`.
- **Авторизация:** JWT (тот же access-token, что и для REST) передаётся **в query-параметре `token`**, не в заголовке. Без валидного токена сервер сразу закрывает соединение.
- Открывать соединение **после** входа; при истечении токена — переоткрыть с новым.

### Жизненный цикл и keepalive
- Сервер каждые **~30 сек** шлёт сообщение `PING`. Клиент должен отвечать `PONG` (см. ниже), иначе соединение считается неактивным.
- **Таймаут бездействия — ~5 минут**: если соединение «молчит», сервер его закрывает. Клиент должен **переподключаться автоматически** (с backoff) при разрыве/потере сети.
- Долгоживущие соединения через nginx разрешены (read/send timeout 1 час).

### Формат сообщений
Все сообщения — JSON одного конверта `WebSocketMessageDTO`. Поле `type` определяет вид:

```jsonc
{
  "type": "EVENT | PING | PONG | SUBSCRIPTION | UNSUBSCRIPTION | ERROR",
  "event":        { ... },   // StationEventDTO — изменения станции/коннектора (broadcast)
  "bookingEvent": { ... },   // BookingEventDTO — прогресс брони (для своего userId)
  "chargingEvent":{ ... },   // ChargingStatusDTO — живые показания зарядки (для своего userId)
  "stationId": "CB-1",       // для подписки/отписки и подтверждений
  "message": "текст",        // для ERROR и ack
  "timestamp": 1719312000000,
  "serverTime": 1719312000123
}
```

### Что шлёт КЛИЕНТ (входящие для сервера)
- **`PONG`** — ответ на `PING` сервера (обязательно для keepalive):
  ```json
  { "type": "PONG" }
  ```
- **`SUBSCRIPTION`** — подписка на конкретную станцию (например, когда открыта её карточка):
  ```json
  { "type": "SUBSCRIPTION", "stationId": "CB-1" }
  ```
- **`UNSUBSCRIPTION`** — отписка (закрыл карточку станции):
  ```json
  { "type": "UNSUBSCRIPTION", "stationId": "CB-1" }
  ```
  > Примечание: события станций сейчас приходят **широковещательно** (broadcast) всем подключённым — подписка нужна как явное указание интереса и подтверждается ack’ом; на неё можно опираться для будущей адресной фильтрации.

### Что шлёт СЕРВЕР (исходящие для клиента)
- **`PING`** — каждые 30 сек, ждёт `PONG`.
- **`EVENT`** — изменение станции/коннектора (поле `event` = `StationEventDTO`):
  - `eventType`: `STATION_CREATED | STATION_UPDATED | STATION_DELETED | CONNECTOR_STATUS_CHANGED | LOCATION_UPDATED | VERSION_INCREMENTED | METER_VALUE | TARIFF_UPDATED`;
  - `stationId`, `stationVersion`, `stationState` (актуальный статус станции/коннекторов), опц. `meterValue`.
  - **Применение в UI:** перекрасить пин/карточку на карте, обновить статус коннектора в карточке станции, обновить тариф.
- **`bookingEvent`** (`BookingEventDTO`) — прогресс брони для текущего пользователя: `eventType` (напр. `RESERVATION_PROGRESS`), `reservationId`, `data` (станция, коннектор, `minutesElapsed`, `remainingBookingMinutes`, `currentCost`, `estimatedEndTime`).
  - **Применение:** таймер брони и её статус на экране бронирования.
- **`chargingEvent`** (`ChargingStatusDTO`) — живые показания **активной зарядки** (только для инициировавшего пользователя): `energyKwh`, `currentCost`, `kwCost`, `maxKwQuantity`, `soc`, `status`, `startedAt`.
  - **Применение:** экран 4.11 «Активная сессия зарядки» — крупные цифры кВт·ч/мощность/стоимость, индикатор процесса, предупреждение о низком балансе.
- **`SUBSCRIPTION` / `UNSUBSCRIPTION`** — подтверждение (ack) на запросы клиента.
- **`ERROR`** — ошибка обработки (поле `message`).

> Важно: `bookingEvent` и `chargingEvent` адресные (приходят только владельцу брони/сессии), а `EVENT` по станциям — широковещательные. Клиент различает их по присутствию соответствующего поля + по `type`.

### Рекомендации клиенту
- Игнорировать неизвестные `type`/поля (форвард-совместимость).
- Один сокет на приложение; раздавать события по экранам внутри клиента (карта, карточка станции, сессия).
- Реалтайм — **дополнение** к REST: исходное состояние тянуть REST-запросом (`/api/cached-stations`), затем накладывать дельты из WebSocket.
- При разрыве — переподключение с экспоненциальным backoff + повторная подписка на открытую станцию.

## 6. Сквозные пользовательские сценарии (happy path)

1. **Первая зарядка:** Открыл приложение → карта показала ближайшие станции → выбрал зелёную → карточка станции → выбрал свободный коннектор → «Забронировать» (или «Начать зарядку») → подтвердил → экран активной сессии с живыми кВт·ч → «Остановить» → итог и списание с баланса.
2. **Пополнение перед зарядкой:** Кошелёк → «Пополнить» → сумма 500 → оплата O!Dengi (QR) → успех → баланс обновился.
3. **Пополнение во время зарядки:** На экране сессии «Пополнить во время зарядки» → оплата → сессия продлевается, лимит кВт·ч увеличивается.
4. **Поиск конкретного разъёма:** Карта → Фильтры → тип CCS + «только свободные» → список → станция → бронь.
5. **Избранное:** На карте тапнул станцию → в превью нажал ♡ (стало ♥) → позже открыл ⭐ «Избранное» → увидел ту же станцию с актуальным расстоянием/статусом → тап → бронь.

## 7. Дизайн-направление (рекомендации, не жёсткие)

- Чистый, «технологичный» стиль; крупные тапабельные элементы; акцентный цвет — зелёный (электро/эко).
- Карта — центр опыта; на ней минимум визуального шума, статусы читаются цветом мгновенно.
- Главные действия — одна основная кнопка на экран (sticky внизу).
- Реалтайм-данные на экране зарядки — крупно и наглядно (числа + анимация).
- Тёмная и светлая темы.
- Иконки разъёмов (Type 2 / CCS / CHAdeMO / GB/T) — отдельный набор.

## 8. Список возможностей бэкенда (для справки, без деталей реализации)

> Эндпоинты приведены списком только чтобы понимать, какие данные доступны. Подробная интеграция — отдельно.

**Аутентификация / пользователь** (`user-service`):
- Регистрация, вход, выход, обновление токена.
- Подтверждение email (запрос/повтор/подтверждение), статус верификации.
- Запрос и сброс пароля.
- Верификация телефона (запрос/подтверждение).
- Профиль: просмотр, редактирование.
- **Избранные станции:** список ID избранного, добавить/удалить станцию (хранится за пользователем; источник истины — БД).

**Станции и коннекторы** (`station-controll-service`):
- Список станций (с пагинацией), детали станции.
- Типы коннекторов, статусы коннекторов.
- Адреса станций.
- Старт/стоп транзакции (зарядки) на коннекторе.

**Геопоиск и кэш** (`state-updater-service`):
- Станции в радиусе, в bounding-box, расчёт расстояния, данные по станции.
- Кэшированные статусы станций (быстрая отдача для карты).
- **Обогащение по списку ID:** отдать данные конкретных станций (по набору ID) с расстоянием от точки — используется для экрана «Избранное», возвращает те же поля, что и общий список.

**Реальное время** (`websocket-service`) — подробности в разделе 5a:
- WebSocket `/.../ws/station-events?token=<JWT>` — push событий станций/коннекторов (broadcast), прогресса брони и живых показаний зарядки (адресно своему пользователю).
- Клиентские сообщения: `PONG` (keepalive), `SUBSCRIPTION`/`UNSUBSCRIPTION` на станцию.

**Бронирование** (`booking-service`):
- Создание брони (сага: проверка баланса + резерв станции), завершение брони.

**Оплата и кошелёк** (`payment-service`):
- Баланс пользователя.
- Пополнение через O!Dengi: инициация, статус заказа, история пополнений.
- (кошелёк создаётся автоматически при регистрации.)

**Зарядка (OCPP-мост)** (`station-integration-service`):
- Удалённый старт/стоп зарядки, список коннекторов.

**Уведомления** (`notification-service`):
- Email-уведомления (подтверждение почты, сброс пароля, неисправность станции).
